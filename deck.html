Vaults & Credentials

Per-user authentication for MCP

Vaults group credentials by user. Each credential binds to an MCP server URL and is automatically injected at session runtime.

Credential Types

OAuth mcp_oauth

Access token + refresh flow. Anthropic handles token rotation.

Bearer static_bearer

Fixed API key or personal access token. No refresh needed.

Codebase

Clean Swift architecture

// Project structure ClaudeCrew/ ├── App/ │ └── ClaudeCrewApp.swift ├── API/ │ ├── AnthropicClient.swift // actor, async/await │ └── KeychainHelper.swift // secure storage ├── Models/ │ ├── Agent.swift // tools, MCP, versioning │ ├── Environment.swift // packages, networking │ ├── Session.swift // status, lifecycle │ ├── Vault.swift // credentials, OAuth │ └── Event.swift // SSE event types ├── ViewModels/ │ ├── AppState.swift // @Observable │ └── SessionViewModel.swift // stream mgmt └── Views/ ├── Agents/ // list, detail, create ├── Environments/ // list, detail, create ├── Sessions/ // list, detail, events ├── Vaults/ // list, detail, creds └── Settings/ // API key management

🎭 Actor-based API Client

AnthropicClient is a Swift actor for safe concurrent access. All network calls are async/await.

📡 SSE Streaming

Uses URLSession.bytes for real-time Server-Sent Events parsing. Wrapped in AsyncThrowingStream.

🔍 @Observable State

Swift Observation framework for reactive UI. No Combine, no manual publishers.

🔒 Keychain Storage

API key goes straight to macOS Keychain via Security framework. Zero disk exposure.
